# Venture Intelligence Layer Skill

## Purpose

Use this skill when FOUNDRYOS needs to think beyond one company and operate like an autonomous venture studio.

## Core Loop

```text
Find signal
Create venture idea
Simulate buyer reaction
Stress test
Create company package
Launch micro-test
Measure demand
Improve template
Scale, pause, or kill
```

## Agents

- Market Radar Agent
- Venture Simulation Agent
- Capital Allocator Agent
- Company Spawner Agent
- Synthetic Customer Council
- Proof Factory Agent
- Founder Narrative Agent
- M&A Radar Agent
- Hive Court Agent

## Market Radar Protocol

Look for:

- repeated buyer pain
- industry complaints
- manual workflows
- platform changes
- new tools and APIs
- job posts that reveal budget
- competitor weakness
- new compliance pressure
- customer requests

Output:

```markdown
# Opportunity Brief

## Signal

## Buyer

## Pain

## Existing workaround

## First offer

## Why now

## Micro-test

## Score
```

## Venture Simulation Protocol

Before building a venture, simulate:

- ICP
- offer
- price
- demo
- objections
- delivery complexity
- cost
- risks
- recurring revenue path
- exit path

Output:

```text
Launch / Modify / Kill
```

## Synthetic Customer Council Protocol

Create 5-7 simulated buyers and skeptics.

Each one must answer:

- Do I understand it?
- Would I pay?
- What scares me?
- What proof do I need?
- What is missing?
- What would make this urgent?

## Company Spawner Protocol

If a venture passes the score threshold, create a package with:

- company thesis
- ICP
- offer
- pricing
- org chart
- agents
- skills
- GTM plan
- sales pitch
- proposal flow
- delivery workflow
- proof path
- licence plan
- exit thesis

## Proof Factory Protocol

Turn every real artifact into reusable proof.

Sources:

- demo
- screenshot
- install log
- proposal
- customer objection
- case study
- revenue event
- skill update
- board report

Outputs:

- landing page section
- sales post
- short video script
- email
- investor update
- pitch slide

## Hive Court Protocol

For important decisions, ask:

- CEO: strategic upside
- CFO: cost and margin
- CRO: buyer demand
- CTO: build complexity
- Risk: governance issues
- Customer Agent: buyer trust
- Investor Agent: valuation impact
- Board Agent: final ruling

Ruling options:

```text
Approve
Reject
Modify
Run experiment
Escalate
```

## Venture Scorecard

Score 1-5:

- buyer pain
- urgency
- budget
- distribution access
- speed to proof
- delivery burden
- margin potential
- recurring revenue potential
- defensibility
- exit potential

Verdict:

```text
0-20: Kill
21-30: Research more
31-40: Micro-test
41-50: Spawn company package
```

## Rule

No venture survives because it sounds cool.

It survives only if it creates proof, demand, revenue, defensibility, or strategic value.
