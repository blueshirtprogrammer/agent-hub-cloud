# HighLevel AI Employee Harness Skill

## Purpose

Use this skill when FOUNDRYOS needs to use HighLevel as a hosted agency execution layer for CRM, workflows, AI Employee, Voice AI, Conversation AI, marketplace app installation, and white-label distribution.

## Harness role

```text
HighLevel = CRM / workflow / telephony / AI Employee / agency distribution
FOUNDRYOS = company templates / venture intelligence / licence / executive governance / capital studio
```

## Approved uses

- create founder install pipelines
- create opportunities
- route leads to calendars
- connect Voice AI agents
- retrieve call logs and transcripts
- trigger workflows
- map templates into sub-accounts
- personalise onboarding using installer context
- support agency white-label distribution

## Build sequence

1. Create HighLevel private Marketplace app.
2. Configure OAuth scopes with least privilege.
3. Add external authentication to connect FOUNDRYOS.
4. Use App Installer Details for agency/sub-account context.
5. Create FOUNDRYOS Founder Installs pipeline.
6. Map company templates to workflows and opportunities.
7. Configure Voice AI/Conversation AI as execution agents.
8. Add licence heartbeat and white-label settings.
9. Test in one agency account.
10. Prepare public Marketplace app listing.

## Guardrails

- Use official OAuth and Marketplace app flows.
- Do not scrape sessions or browser tokens.
- Do not bypass billing, usage limits, app review, or fair-use rules.
- Disclose SaaS/licence dependency clearly.
- Use least privilege scopes.
- Route sensitive actions through the Executive Approval Chain.
