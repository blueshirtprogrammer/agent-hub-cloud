# Self-Launch Autopilot Skill

## Purpose

Use this skill to make FOUNDRYOS launch and sell itself with minimal human labour and strict approval gates.

## Mission

Move FOUNDRYOS through this sequence:

```text
public URL live
→ demo created
→ content produced
→ targeted prospects identified
→ personalised outreach drafted
→ approved outreach sent
→ calls booked
→ 3 founder installs sold
→ case studies created
→ installs converted to monthly leases
→ licence heartbeat enforced
```

## Inputs

- public URL status
- current repo state
- available company templates
- current offer
- target ICP
- available channels
- approval settings
- CRM or prospect list
- payment/licence status

## Outputs

- launch checklist
- demo script
- content calendar
- social posts
- prospect list
- personalised outreach drafts
- booking CTA
- proposal draft
- payment link request
- case study draft
- lease conversion plan
- licence heartbeat status
- board report

## Daily Protocol

1. Check whether the public URL is live.
2. If not live, create deployment task and escalation.
3. If live, produce or improve demo asset.
4. Repurpose demo into social/email assets.
5. Identify 25 high-fit prospects.
6. Draft personalised outreach.
7. Queue public/external actions for approval.
8. Send only approved outreach.
9. Triage replies.
10. Book calls.
11. Draft proposal/payment link.
12. Create proof and update board report.
13. Identify one skill/process improvement.

## Public Action Approval Gates

Require approval for:

- public posts
- outbound campaigns
- paid ads
- proposals
- payment links
- contracts
- customer claims
- testimonials
- revenue claims
- licence suspension logic
- production deploy changes
- credential changes

## Content Asset Factory

Turn one demo into:

- 1 long LinkedIn post
- 5 short posts
- 3 short video scripts
- 1 email
- 1 one-page sales asset
- 1 case study shell
- 1 investor update

## Case Study Rules

Never invent customer outcomes.

Use:

- before state
- deployment steps
- screenshots
- artifacts
- customer-approved quote
- measurable result
- public/private version

## Licence Loop

If licence is active:

- allow premium templates
- allow agent heartbeat runtime
- allow premium skills
- allow revenue autopilot

If licence is past due:

- show warning
- pause premium agent heartbeats after grace period
- keep data/export access
- do not delete customer data
