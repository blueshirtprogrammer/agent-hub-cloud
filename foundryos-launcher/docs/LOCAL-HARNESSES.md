# Local Harness Setup

FOUNDRYOS Launcher treats IDE apps and coding agents as business execution harnesses.

## Harnesses

- Paperclip: agent company control plane
- Codex: repo execution and product engineering
- OpenCode: local skill-driven operator
- Kilo Code: multi-IDE engineering agent
- Antigravity: browser workflow verification and proof capture
- OpenRouter: model routing layer
- GitHub: source of truth, commits, issues, PRs, audit trail
- GHL: CRM and revenue ops
- Stripe/Xero: billing and finance ops

## Manual setup principle

Install each harness from its official source and connect credentials using official OAuth, API keys, or CLI auth.

Do not automate credential extraction, browser-cookie reuse, fake account creation, or platform-limit bypasses.

## Runtime standard

Every harness action must produce proof:

- file path
- commit or PR
- screenshot
- browser recording
- CRM record
- proposal draft
- invoice draft
- deployment URL
- board report
