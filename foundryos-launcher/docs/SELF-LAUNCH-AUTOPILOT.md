# FOUNDRYOS™ Self-Launch Autopilot

## Purpose

This is the operating layer that lets FOUNDRYOS launch, sell, demonstrate, report, and monetise itself with minimal human labour.

The goal is not full reckless autonomy.

The goal is:

```text
Maximum autonomous execution
+ clean credentials
+ public-action approval gates
+ measurable revenue movement
```

## Core outcome

FOUNDRYOS must be able to run this loop:

```text
Get public URL live
→ create 5-minute demo
→ generate content assets
→ publish approved content
→ identify buyers
→ personalise outreach
→ book calls
→ sell 3 founder installs
→ create case studies
→ convert installs to monthly leases
→ enforce licence entitlements
→ improve its own skills
```

## Agent responsibilities

## 1. Deployment Agent

Owns:

- Vercel project setup guide
- Fly.io runtime deployment guide
- public URL capture
- DNS/custom domain checklist
- uptime checks
- deployment proof

Can execute:

- repo config updates
- deployment docs
- health-check tasks
- public URL capture once available

Requires approval for:

- adding production secrets
- changing billing infra
- destructive deployment changes

## 2. Demo Producer Agent

Owns:

- 5-minute demo script
- shot list
- screen capture guide
- narration script
- demo CTA
- demo landing page embed

Outputs:

- video script
- scene plan
- captions
- thumbnail copy
- description
- short clips

## 3. AI Video Agent

Owns AI avatar/video creation workflows.

Approved tools:

- HeyGen
- Synthesia
- Descript
- Canva
- browser-based editors

Rules:

- only use authorised avatars/voices/likenesses
- no impersonation
- no fake customer testimonials
- no undisclosed synthetic human where disclosure is required

## 4. AI Voice Agent

Owns voiceover and audio assets.

Approved tools:

- ElevenLabs
- native TTS
- Descript
- provider-approved voice cloning only

Rules:

- only clone voices with explicit consent
- disclose synthetic voice where appropriate
- never impersonate real people without approval

## 5. Social Strategist Agent

Owns content strategy.

Channels:

- LinkedIn
- X
- YouTube Shorts
- TikTok
- email newsletter
- founder DMs with approval

Outputs:

- daily content calendar
- campaign angles
- hooks
- proof posts
- founder-led narratives
- audience-specific CTAs

Requires approval for:

- publishing to public channels
- mass outreach
- claims about customers/revenue

## 6. Content Repurposing Agent

Turns one demo into:

- 1 long post
- 5 short posts
- 3 short videos
- 1 email
- 1 sales page section
- 1 case study skeleton
- 1 investor update

## 7. Prospecting Agent

Builds target lists.

First ICPs:

- AI agency owners
- GHL agencies
- automation consultants
- hospitality consultants
- venture studios
- enterprise innovation operators

Data fields:

- name
- company
- role
- niche
- pain hypothesis
- trigger
- channel
- source
- confidence score
- next action

## 8. Personalisation Agent

Creates targeted outreach.

Rules:

- no fake familiarity
- no false claims
- no scraping behind auth walls
- no spam blasts
- no pretending to be human if the channel requires disclosure

Outputs:

- personalised opener
- pain hypothesis
- relevant offer
- CTA
- follow-up sequence

## 9. SDR / Booking Agent

Owns:

- approved outreach sending
- reply triage
- meeting booking
- CRM updates
- no-show follow-ups

Requires approval for:

- first outbound campaign
- volume increase
- new audience segment
- automated replies to sensitive prospects

## 10. Sales Closer Agent

Owns:

- discovery script
- demo flow
- objection handling
- close plan
- proposal draft
- payment link draft

Requires approval for:

- sending proposal
- sending payment link
- discounting
- accepting custom scope

## 11. Case Study Agent

Turns installs into proof.

Outputs:

- before/after
- problem
- deployment steps
- screenshots
- metrics
- quote request
- public/private version

Rules:

- no fake results
- no invented customer quotes
- customer approval required for public naming

## 12. Subscription Conversion Agent

Turns founder installs into monthly leases.

Offer path:

```text
Founder Install
→ 14-day operating proof
→ monthly company lease
→ premium skills/harness updates
→ enterprise private foundry
```

## 13. Licence Entitlements Agent

Owns paid access logic.

Integrates with:

- Stripe Subscriptions
- Stripe Entitlements
- internal licence table
- heartbeat endpoint
- feature flags

Rules:

- never delete customer data for non-payment
- gracefully disable premium features
- keep export available
- honour grace periods

## 14. Analytics / Benchmark Agent

Measures:

- public URL live status
- demo views
- content posts shipped
- replies
- booked calls
- install payments
- active leases
- churn risk
- MRR
- ARR
- CAC
- time-to-install
- agent task success rate

## Self-launch operating loop

### Daily loop

```text
1. Check public URL status
2. Check licence/payment status
3. Review pipeline
4. Create/repurpose one proof asset
5. Identify 25 high-fit prospects
6. Draft personalised outreach
7. Queue public/social actions for approval
8. Send approved outreach
9. Triage replies
10. Book calls
11. Draft proposals/payment links
12. Update dashboard
13. Improve skills
```

### Weekly loop

```text
1. Publish demo/proof update
2. Run YC stress-test skill
3. Review MRR/ARR and install pipeline
4. Update case studies
5. Improve template/product gaps
6. Decide next ICP or offer change
```
