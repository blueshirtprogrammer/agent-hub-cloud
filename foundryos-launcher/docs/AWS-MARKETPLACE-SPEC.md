# FOUNDRYOS AWS Marketplace Spec

## Purpose

Prepare FOUNDRYOS for enterprise procurement through AWS Marketplace.

## Candidate products

```text
FOUNDRYOS Private Foundry SaaS
FOUNDRYOS Launcher Container
FOUNDRYOS Enterprise Deployment Package
FOUNDRYOS Venture Studio Operating Layer
```

## Listing path options

### SaaS listing

Best for:

- hosted control plane
- enterprise subscription
- private foundry
- licence heartbeat
- Stripe alternative via AWS billing

### Container product

Best for:

- self-hosted launcher
- customer-owned deployment
- private cloud / AWS-first teams

### Professional services wrapper

Best for:

- enterprise setup
- private foundry implementation
- integration and onboarding

## Readiness checklist

- seller registration
- support process
- fulfilment landing page
- architecture diagram
- security overview
- pricing dimensions
- customer onboarding flow
- subscription status in app
- refund/support policy
- deployment model
- data handling notes

## Fulfilment landing page

Must collect:

```text
buyer email
company name
preferred deployment type
selected package
technical contact
billing/contact notes
```

Must show:

```text
subscription status
next setup step
support link
onboarding instructions
```

## Pricing dimensions

Potential dimensions:

```text
company_lease
premium_templates
agent_runtime
enterprise_private_foundry
studio_portfolio_seats
```

## Guardrails

- SaaS listing must use marketplace billing dimensions.
- Customers must be able to access the SaaS app after subscription.
- Support contact must be clear.
- Any buyer infrastructure requirements must be disclosed.
- Use least privilege for AWS deployment templates.
