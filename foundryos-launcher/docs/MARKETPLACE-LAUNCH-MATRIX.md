# FOUNDRYOS Marketplace Launch Matrix

## Purpose

This matrix turns FOUNDRYOS into a multi-channel distribution machine.

## Immediate channel priorities

| Channel | Product | First asset | Status |
|---|---|---|---|
| Vercel | Public launcher URL | Live app deploy | Priority 1 |
| Product Hunt | FOUNDRYOS launch | Demo, screenshots, maker comment | Priority 1 |
| Docker Hub | foundryos/launcher | Image, tags, run command | Priority 1 |
| npm | @foundryos/launcher | CLI/package scaffold | Priority 1 |
| HighLevel | Agency Company Launcher | Private app, OAuth, white-label app | Priority 2 |
| WordPress | FOUNDRYOS Launcher plugin | Plugin scaffold, admin settings, embed | Priority 2 |
| Skills | FOUNDRYOS skill packs | SKILL.md bundles | Priority 2 |
| AWS Marketplace | Private Foundry SaaS/container | Seller readiness, architecture, fulfilment | Priority 3 |
| Affiliates | Founder install referral program | Partner page, terms, swipe copy | Priority 1 |

## Channel readiness checklist

### Product Hunt

- public URL
- product tagline
- 5-minute demo
- 60-second demo
- screenshots
- maker comment
- FAQ pack
- founding install offer
- launch-day social posts

### Docker Hub

- Dockerfile
- docker-compose.yml
- image name
- tags
- run command
- security note
- upgrade path

### npm

- package name
- README
- install command
- CLI command
- TypeScript support
- examples
- versioning

### WordPress

- plugin folder
- readme.txt
- admin settings page
- licence key field
- launcher embed shortcode/block
- SaaS disclosure
- privacy disclosure
- support link

### HighLevel Marketplace

- developer account
- private app first
- OAuth scopes
- agency/sub-account target
- white-label listing
- install flow
- external auth if needed
- app installer details usage

### AWS Marketplace

- seller registration
- support process
- architecture diagram
- fulfilment landing page
- pricing dimensions
- customer subscription status view
- SaaS/container decision
- security model

### Affiliate Program

- partner landing page
- referral tracking
- commission model
- founder install swipe copy
- monthly lease rev-share
- enterprise intro fee

## First 7-day execution

```text
Day 1: public URL and Product Hunt launch assets
Day 2: Docker image and npm package scaffold
Day 3: affiliate founder install page
Day 4: HighLevel private app spec and OAuth flow
Day 5: WordPress plugin scaffold
Day 6: skill marketplace bundle

```

## Rule

Every distribution channel must connect back to one of:

- founder install
- company lease
- enterprise private foundry
- studio deal intake
- strategic partnership
