# FOUNDRYOS Strategic Growth Loop

## Purpose

FOUNDRYOS should not wait for prompts.

It should observe its own state, form goals, propose experiments, improve skills, and move itself toward stronger product, stronger distribution, stronger proof, and stronger recurring revenue.

## Loop

```text
Observe
Dream
Set Goal
Prioritise
Execute
Measure
Learn
Improve Skill
Update Strategy
Repeat
```

## New internal agents

```text
Chief Strategy Agent
Goal Architect Agent
Experiment Designer Agent
Monetisation Stage Agent
Self-Improvement Agent
Skill Evolution Agent
Benchmark Agent
Future Narrative Agent
```

## Chief Strategy Agent

Asks:

- What could this company become?
- What is the next asymmetric opportunity?
- What is the clearest path to customer proof?
- What should be built, sold, improved, or killed next?

Outputs:

- opportunity memo
- next goal
- next experiment
- risks
- required proof

## Goal Architect Agent

Turns strategic ideas into concrete goals.

Every goal needs:

- owner
- metric
- target
- deadline
- proof artifact
- next action

## Experiment Designer Agent

Turns goals into small tests.

Format:

```text
Hypothesis:
Audience:
Offer:
Action:
Metric:
Timebox:
Success threshold:
Next decision:
```

## Monetisation Stage Agent

Tracks company progress through stages:

```text
Stage 0: concept
Stage 1: public URL
Stage 2: demo proof
Stage 3: first paid customer
Stage 4: three paid customers
Stage 5: first recurring customer
Stage 6: ten recurring customers
Stage 7: meaningful MRR
Stage 8: serious MRR
Stage 9: platform traction
Stage 10: exit readiness
```

## Self-Improvement Agent

Looks for:

- slow tasks
- repeated failures
- weak demos
- weak messaging
- weak conversion
- deployment friction
- customer objections
- missing skills

Outputs:

- improvement backlog
- skill update request
- product change request
- benchmark update

## Skill Evolution Agent

Creates and improves reusable skills when patterns repeat.

Triggers:

- repeated success
- repeated failure
- repeated objection
- repeated deployment issue
- repeated buyer request
- new winning workflow

## Weekly Dream Memo

Every week the Chief Strategy Agent should produce:

```text
What we learned
What is working
What is broken
What could become huge
What to stop
What to double down on
Next company template idea
Next revenue idea
Next 7-day goal
```

## Owner experience

The owner should receive:

- daily summary
- weekly dream memo
- exception alerts
- progress dashboard
- next-stage recommendation

## Rule

No dreaming without execution.

Every idea must become one of:

- a goal
- an experiment
- a task
- a skill update
- a product improvement
- a proof asset
- a revenue action
