# FOUNDRYOS Venture Intelligence Layer

## Purpose

FOUNDRYOS should not only improve one company.

It should discover new opportunities, simulate venture ideas, create new company packages, test demand, allocate effort, compound proof, and prepare the strongest ventures for scale or exit.

## Core loop

```text
Find signal
Create venture idea
Simulate buyer reaction
Stress test
Create company package
Launch micro-test
Measure demand
Improve template
Scale, pause, or kill
```

## New internal agents

```text
Market Radar Agent
Venture Simulation Agent
Capital Allocator Agent
Company Spawner Agent
Synthetic Customer Council
Proof Factory Agent
Founder Narrative Agent
M&A Radar Agent
Hive Court Agent
```

## Market Radar Agent

Watches for opportunity signals.

Signals include:

- buyer complaints
- fast-growing niches
- new tools and APIs
- platform shifts
- job posts showing pain
- regulatory change
- competitor gaps
- repeated customer requests
- new pricing patterns
- manual workflows that should become products

Output:

```text
Opportunity Brief
```

## Venture Simulation Agent

Simulates a company before building it.

Creates:

- offer
- landing page outline
- pricing
- target customer
- sales pitch
- likely objections
- demo concept
- delivery burden
- risk map
- exit logic

Output:

```text
Launch / Modify / Kill
```

## Capital Allocator Agent

Allocates company effort and budget.

Inputs:

- available time
- available cash
- current bottleneck
- expected upside
- confidence
- risk
- speed to proof

Output:

```text
This week allocation
```

## Company Spawner Agent

Creates new company packages.

Each package includes:

- company thesis
- ICP
- offer
- pricing
- agent org chart
- skill list
- GTM sprint
- proposal flow
- invoice flow
- delivery flow
- proof path
- licence plan
- exit thesis

## Synthetic Customer Council

Creates a panel of simulated buyers and skeptics.

Examples:

- AI agency owner
- hotel GM
- venture studio partner
- enterprise innovation lead
- automation consultant
- CFO buyer
- risk/compliance skeptic

Each council member attacks the offer before it reaches the market.

## Proof Factory Agent

Turns operating history into proof assets.

Sources:

- screenshots
- demo clips
- install logs
- proposals
- customer objections
- case studies
- revenue events
- board reports
- skill updates

Outputs:

- landing page sections
- sales posts
- investor updates
- demo snippets
- case studies
- pitch slides

## Founder Narrative Agent

Builds founder-led content and positioning.

Outputs:

- LinkedIn posts
- X posts
- video scripts
- launch essays
- sales emails
- investor updates
- founder POVs

Rule:

No generic AI slop. Strong opinion, clear market insight, proof-led narrative.

## M&A Radar Agent

Maps strategic exit paths.

For each venture it asks:

- Who would buy this?
- Why would they buy it?
- What proof would they need?
- What revenue level matters?
- What asset compounds?
- What integrations increase acquirer fit?

## Hive Court Agent

Runs internal debates for important decisions.

Participants:

- CEO Agent
- CFO Agent
- CRO Agent
- CTO Agent
- Legal/Risk Agent
- Customer Agent
- Investor Agent
- Board Agent

Possible rulings:

```text
Approve
Reject
Modify
Run experiment
Escalate
```

## Venture scorecard

Score every venture idea from 1-5 across:

- buyer pain
- urgency
- budget
- distribution access
- speed to proof
- delivery burden
- margin potential
- recurring revenue potential
- defensibility
- exit potential

Interpretation:

```text
0-20: Kill
21-30: Research more
31-40: Micro-test
41-50: Spawn company package
```

## Owner experience

The owner does not need to invent every next move.

The owner receives:

- best opportunities
- current venture scores
- next experiments
- resource allocation summary
- scale / pause / kill recommendations
- cash and proof dashboard

## Rule

No venture survives because it sounds cool.

It survives only if it creates proof, demand, revenue, defensibility, or strategic value.
